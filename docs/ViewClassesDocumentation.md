# Документация классов слоя Представления (View)

## Общее описание

Классы представления отвечают за отображение данных пользователю и обработку пользовательских действий. Каждый класс представления отвечает за свой блок разметки и не хранит в себе данных - вся логика хранения данных находится в моделях.

## Иерархия классов представления

### Базовые классы

#### View<T>
Базовый абстрактный класс для всех компонентов представления. Наследуется от [Component](file:///c%3A/Users/%D0%B0%D1%80%D1%82/Desktop/%D1%80%D0%B0%D0%B1%D0%BE%D1%87%D0%B8%D0%B9%20%D1%81%D1%82%D0%BE%D0%BB/weblarek/src/components/base/Component.ts#L3-L26) и предоставляет доступ к брокеру событий.

- **Конструктор**: Принимает контейнер HTMLElement и объект IEvents для работы с событиями
- **Поля**: 
  - `container`: HTMLElement - корневой элемент компонента
  - `events`: IEvents - брокер событий

#### BaseCard
Базовый класс для карточек товаров, обеспечивает общую функциональность для всех типов карточек.

- **Конструктор**: Принимает контейнер HTMLElement и объект IEvents
- **Поля**:
  - `_id`: string - идентификатор карточки
  - `titleElement`: HTMLElement - элемент заголовка
  - `imageElement`: HTMLImageElement - элемент изображения
  - `categoryElement`: HTMLElement - элемент категории
  - `priceElement`: HTMLElement - элемент цены
  - `descriptionElement`: HTMLElement - элемент описания
- **Методы**:
  - `id`: сеттер для установки ID карточки
  - `title`: сеттер для установки заголовка
  - `image`: сеттер для установки изображения
  - `category`: сеттер для установки категории (применяет CSS-классы из categoryMap)
  - `price`: сеттер для установки цены (отображает "Недоступно" при null)
  - `description`: сеттер для установки описания

#### Form<T>
Абстрактный базовый класс для всех форм, обеспечивает общую функциональность для форм.

- **Конструктор**: Принимает контейнер HTMLElement и объект IEvents
- **Поля**:
  - `inputs`: HTMLInputElement[] - массив всех input элементов
  - `submitButton`: HTMLButtonElement - кнопка отправки формы
- **Методы**:
  - `formName`: абстрактное свойство для получения названия формы
  - `setFieldValue`: устанавливает значение поля формы
  - `setSubmitState`: устанавливает состояние кнопки отправки
  - `setErrors`: устанавливает ошибки валидации

### Классы карточек (наследуются от BaseCard)

#### ProductCard
Карточка товара для отображения в каталоге на главной странице.

- **Конструктор**: Принимает контейнер HTMLElement и объект IEvents
- **Поля**:
  - `_title`: HTMLElement - элемент заголовка
  - `_image`: HTMLImageElement - элемент изображения
  - `_category`: HTMLElement - элемент категории
  - `_price`: HTMLElement - элемент цены
  - `_description`: HTMLElement - элемент описания
  - `_button`: HTMLButtonElement - кнопка покупки
- **Методы**:
  - `id`: геттер/сеттер для ID карточки
  - `title`: сеттер для заголовка
  - `image`: сеттер для изображения
  - `category`: сеттер для категории (применяет CSS-классы)
  - `price`: сеттер для цены (блокирует кнопку при null)
  - `description`: сеттер для описания
- **События**:
  - `product:select`: генерируется при клике на кнопку покупки

#### PreviewCard
Карточка товара для предварительного просмотра в модальном окне.

- **Конструктор**: Принимает контейнер HTMLElement и объект IEvents
- **Поля**:
  - `_id`: string - идентификатор карточки
  - `_title`: HTMLElement - элемент заголовка
  - `_image`: HTMLImageElement - элемент изображения
  - `_category`: HTMLElement - элемент категории
  - `_price`: HTMLElement - элемент цены
  - `_description`: HTMLElement - элемент описания
  - `button`: HTMLButtonElement - кнопка действия
- **Методы**:
  - `id`: геттер/сеттер для ID карточки
  - `title`: сеттер для заголовка
  - `image`: сеттер для изображения
  - `category`: сеттер для категории (применяет CSS-классы)
  - `price`: сеттер для цены
  - `description`: сеттер для описания
  - `setInBasketState`: устанавливает состояние кнопки (купить/удалить из корзины)
  - `setDisabledState`: блокирует кнопку
  - `setOnClick`: устанавливает обработчик клика по кнопке

#### BasketItem
Карточка товара в корзине.

- **Конструктор**: Принимает контейнер HTMLElement и объект IEvents
- **Поля**:
  - `_title`: HTMLElement - элемент заголовка
  - `_price`: HTMLElement - элемент цены
  - `_index`: HTMLElement - номер позиции в корзине
  - `_deleteButton`: HTMLButtonElement - кнопка удаления
- **Методы**:
  - `id`: геттер/сеттер для ID карточки
  - `title`: сеттер для заголовка
  - `price`: сеттер для цены
  - `index`: сеттер для индекса
- **События**:
  - `basket:remove`: генерируется при клике на кнопку удаления

### Классы форм (наследуются от Form<T>)

#### ContactsForm
Форма контактной информации покупателя.

- **Конструктор**: Принимает контейнер HTMLElement и объект IEvents
- **Поля**:
  - `_emailInput`: HTMLInputElement - поле электронной почты
  - `_phoneInput`: HTMLInputElement - поле телефона
- **Методы**:
  - `formName`: возвращает "contacts"
  - `email`: сеттер для электронной почты
  - `phone`: сеттер для телефона
- **События**:
  - `contacts:submit`: генерируется при отправке формы
  - `contacts:change`: генерируется при изменении полей формы

#### OrderForm
Форма оформления заказа.

- **Конструктор**: Принимает контейнер HTMLElement и объект IEvents
- **Поля**:
  - `_paymentButtons`: NodeListOf<HTMLInputElement> - кнопки способа оплаты
  - `_addressInput`: HTMLInputElement - поле адреса доставки
- **Методы**:
  - `formName`: возвращает "order"
  - `payment`: сеттер для способа оплаты (обновляет активные классы кнопок)
  - `address`: сеттер для адреса
- **События**:
  - `order:payment:changed`: генерируется при изменении способа оплаты
  - `order:change`: генерируется при изменении полей формы

### Классы компонентов

#### Modal
Компонент модального окна.

- **Конструктор**: Принимает контейнер HTMLElement и объект IEvents
- **Поля**:
  - `_closeButton`: HTMLButtonElement - кнопка закрытия
  - `_content`: HTMLElement - контейнер для содержимого
- **Методы**:
  - `content`: сеттер для установки содержимого модального окна
  - `toggleModal`: показывает/скрывает модальное окно
  - `render`: отображает модальное окно
- **События**:
  - `modal:close`: генерируется при закрытии модального окна

#### GalleryView
Компонент для отображения списка карточек (каталог).

- **Конструктор**: Принимает контейнер HTMLElement и объект IEvents
- **Поля**:
  - `catalogElement`: HTMLElement - контейнер для карточек
- **Методы**:
  - `items`: сеттер для установки списка карточек

#### BasketView
Компонент корзины покупок.

- **Конструктор**: Принимает контейнер HTMLElement и объект IEvents
- **Поля**:
  - `basketList`: HTMLElement - список товаров в корзине
  - `totalElement`: HTMLElement - элемент общей суммы
  - `button`: HTMLButtonElement - кнопка оформления заказа
- **Методы**:
  - `items`: сеттер для установки списка товаров
  - `total`: сеттер для установки общей суммы
  - `setOrderButtonState`: устанавливает состояние кнопки оформления заказа
- **События**:
  - `order:open`: генерируется при клике на кнопку оформления заказа

## События, генерируемые в приложении

### В Моделях данных:
- `items:changed` - изменение каталога товаров
- `item:selected` - изменение выбранного для просмотра товара
- `basket:changed` - изменение содержимого корзины
- `order:changed` - изменение данных покупателя

### В Представлениях:
- `product:select` - выбор карточки для просмотра
- `product:buy` - нажатие кнопки покупки товара
- `basket:remove` - нажатие кнопки удаления товара из корзины
- `basket:open` - нажатие кнопки открытия корзины
- `order:open` - нажатие кнопки оформления заказа
- `order:next` - нажатие кнопки перехода ко второй форме оформления заказа
- `order:submit` - нажатие кнопки оплаты/завершения оформления заказа
- `form:change` - изменение данных в формах

## Особенности реализации

1. Все элементы разметки находятся в конструкторе и сохраняются в полях класса
2. Классы представления не хранят никаких данных - только элементы DOM
3. Все слушатели событий устанавливаются один раз в конструкторе
4. Метод [render()](file:///c%3A/Users/%D0%B0%D1%80%D1%82/Desktop/%D1%80%D0%B0%D0%B1%D0%BE%D1%87%D0%B8%D0%B9%20%D1%81%D1%82%D0%BE%D0%BB/weblarek/src/components/base/Component.ts#L23-L26) вызывается без аргументов и возвращает разметку класса
5. Классы представления не содержат бизнес-логики
6. Модальное окно использует модификатор 'modal_active' для включения/выключения видимости
7. Карточки товаров используют модификаторы из [categoryMap](file:///c%3A/Users/%D0%B0%D1%80%D1%82/Desktop/%D1%80%D0%B0%D0%B1%D0%BE%D1%87%D0%B8%D0%B9%20%D1%81%D1%82%D0%BE%D0%BB/weblarek/src/utils/constants.ts#L15-L21) для отображения фона категорий
8. Форма выбора оплаты использует модификатор 'button_alt-active' для выделения активной кнопки
